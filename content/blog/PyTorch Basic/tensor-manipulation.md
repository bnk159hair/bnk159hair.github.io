    ## 1. 벡터, 행렬 그리고 텐서
    (Vector, Matrix and Tensor)

    ---

    ### 1) 벡터, 행렬, 텐서 그림으로 이해하기

    <p align="center">
            <img src="assets\2021-09-22\00.png"/>
    </p>

    딥 러닝을 다룰 때에 가장 기본적인 단위는 **벡터, 행렬, 텐서**이다. 

    - 차원이 없는 값을 스칼라(위의 그림에는 없음)
    - 1차원으로 구성된 값을 벡터(Vector)
    - 2차원으로 구성된 값을 행렬(Matrix)
    - 3차원 이상이 되면 텐서(Tensor)라고 부른다.

    [데이터 사이언스 분야 한정으로 3차원 이상의 텐서는 그냥 다차원 행렬 또는 배열로 간주할 수 있다. 또한 주로 3차원 이상을 텐서라고 하긴 하지만, 1차원 벡터나 2차원 행렬도 텐서라고 표현하기도 한다.]

    ### 2) Pytorch Tensor Shape Convention

    딥러닝을 할 때 다루고 있는 행렬 또는 텐서의 크기를 고려하는 것은 항상 중요하다. 따라서 행렬과 텐서의 크기를 표현하는 방법을 알아보겠다.

    - **2D Tensor(Typical Simple Setting)**

        **|t| = (Batch size, dim)**

        <p align="center">
            <img src="assets\2021-09-22\01.png"/>
    </p>

        위의 경우는 2차원 텐서의 크기 **|t|를 (batch size x dimension)**으로 표현하였을 경우이다. 아래의 그림과 같이 행렬에서 **행의 크기가 batch size**, **열의 크기가 dim**이라는 의미다.

        <p align="center">
            <img src="assets\2021-09-22\02.png"/>
    </p>

        - 훈련 데이터 하나의 크기가 256일 때, [6, 1, 5, ...] 와 같은 배열의 길이가 256이라고 보면 된다. 
        다시 말해 **훈련 데이터 하나 = 벡터의 차원은 256**이다.
        - 만약 이런 훈련 데이터의 개수가 3000개라고 하면 현재 훈련 데이터의 크기는 3000 x 256이다.
        - 컴퓨터는 훈련 데이터를 하나씩 처리하는 것보다 보통 **덩어리로 처리**하는데 **64개씩 처리한다고 한다면** 이 때 **batch size를 64**라고 한다. 
        → **컴퓨터가 한 번에 처리하는 2D 텐서의 크기는 (batch size x dim) = 64 x 256이다.**
        - **하나의 데이터의 크기= 데이터의 차원이다. 
        [X] $Data = [x] (=x_1, x_2, x_3,..., x_n)$이 아니라 
        [O] $Data = [x_1, x_2, x_3,..., x_n]$  이다. 따라서 n개의 차원 즉,
        하나의 데이터의 크기(n)은 데이터의 차원이라 할 수 있는 것이다.**
    - **3D Tensor(Typical Computer Vision) - 비전 분야에서의 3차원 텐서**

        |t| = (batch size, width, height)

        <p align="center">
            <img src="assets\2021-09-22\03.png"/>
    </p>

        **비전분야(이미지, 영상 처리)**를 하게 되면 좀 더 복잡한 텐서를 다루게 된다. **이미지라는 것은 가로, 세로가 존재한다.** 
        그리고 여러 장의 이미지, 그러니까 batch size로 구성하게 되면 위와 같은 **3차원의 텐서**가 된다.
        위의 그림은 **세로는 batch size, 가로는 너비(width), 그리고 안쪽으로는 (height)**가 되는 것을 볼 수 있다. 

    - **3D Tensor(Typical Natural Language Processing) - NLP 분야에서의 3차원 텐서**

        |t| = (batch size, length, dim)자연어 처리는 보통 (batch size, 문장 길이, 단어 벡터의 차원)이라는 3차원 텐서를 사용한다.

        <p align="center">
            <img src="assets\2021-09-22\04.png"/>
    </p>

        - **NLP 분야의 3D 텐서 예재로 이해하기**

            아래와 같이 4개의 문장으로 구성된 전체 훈련 데이터가 있다하자.

            ```python
            [[나는 사과를 좋아해], [나는 바나나를 좋아해],
             [나는 사과를 싫어해], [나는 바나나를 싫어해]]
            ```

            컴퓨터는 다음과 같이 데이터를 입력받았을 때에 '나는 사과를 좋아해'가 단어가 1개인지 3개인지 이해하지 못하기에 단어별로 나눠줘야한다.

            ```python
            [['나는', '사과를', '좋아해'], ['나는', '바나나를', '좋아해'],
             ['나는', '사과를', '싫어해'], ['나는', '바나나를', '싫어해']]
            ```

            이제 훈련 데이터의 크기는 4 x 3의 크기를 가지는 2D 텐서이다. 컴퓨터는 텍스트보다 숫자를 더 잘 처리할 수 있기에 각 단어를 벡터로 만들 것이다.  아래의 기준을 통해 단어를 3차원의 벡터로 변환했다 해보자.

            ```python
            '나는' = [0.1, 0.2, 0.9]
            '사과를' = [0.3, 0.5, 0.1]
            '바나나를' = [0.3, 0.5, 0.2]
            '좋아해' = [0.7, 0.6, 0.5]
            '싫어해' = [0.5, 0.6, 0.7]
            ```

            ```python
            [[[0.1, 0.2, 0.9], [0.3, 0.5, 0.1], [0.7, 0.6, 0.5]],
             [[0.1, 0.2, 0.9], [0.3, 0.5, 0.2], [0.7, 0.6, 0.5]],
             [[0.1, 0.2, 0.9], [0.3, 0.5, 0.1], [0.5, 0.6, 0.7]],
             [[0.1, 0.2, 0.9], [0.3, 0.5, 0.2], [0.5, 0.6, 0.7]]]
            ```

            이제 훈련 데이터는 4 x 3 x 3의 크기를 가지는 3D 텐서이다. 이제 batch size를 2로 해보겠다.

            ```python
            첫번째 배치 #1
            [[[0.1, 0.2, 0.9], [0.3, 0.5, 0.1], [0.7, 0.6, 0.5]],
             [[0.1, 0.2, 0.9], [0.3, 0.5, 0.2], [0.7, 0.6, 0.5]]]

            두번째 배치 #2
            [[[0.1, 0.2, 0.9], [0.3, 0.5, 0.1], [0.5, 0.6, 0.7]],
             [[0.1, 0.2, 0.9], [0.3, 0.5, 0.2], [0.5, 0.6, 0.7]]]
            ```

            컴퓨터는 배치 단위로 가져가서 연산을 수행한다. 각 배치의 텐서의 크기는 2 x 3 x 3이다. 이는 (batch size, 문장 길이, 단어 벡터의 차원)의 크기이다.
